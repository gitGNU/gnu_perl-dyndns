What is a Perl DDNS project?

First, Perl is a language for getting your job done. Developed by Larry Wall, who can be considered the most famous language hacker for the web age. Almost every little utility can be best and easiest written using odd-looking Perl language. Perl is also a cross platform language, meaning that you can run the programs under Unix, Linux and Windows environments. Learn more about Perl at http://www.perl.org and Windows Perl downloads are available at http://www.activestate.com

This page describes Perl Dynamic DNS (DDNS) Client Project. Program dyndns.pl is cross platfom client that you can use in any environment that run Perl 5.004 or newer. It works with standard perl and requires no addititional libraries. The program will prevent updating same IP address twice. Ideal for Win32 and Linux dial up scripts. (Linux users, please contact maintainer if your Linux flavor is not covered yet).

Supported dynamic DNS providers:

  • http://www.dydns.org/ Supports 2001-06 new login specification and all Static, dynamic and web host features.
  • http://www.no-ip.com/

Supported OS platforms:

  • Win32 native (Windows 9x/NT/2000/XP/*)
  • Win32 Cygwin (Unix like)
  • Linux (Redhat, Debian). Other distributions not tested. Should work with any FHS compliant Linux distributions as well provided that ifconfig(1) is available.

In short, call the program from your dial up boot sequence with: 

        dyndns.pl --Provider dyndns --login LOGIN --pass PASSWORD --Host yourhost.dyndns.org
